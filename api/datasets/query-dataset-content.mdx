---
title: Query Dataset Content
openapi: post /datasets/{dataset_id}/content/query
---

This endpoint allows you to search and filter the content of a specific dataset using advanced query parameters. It provides more flexibility than the Get Dataset Content endpoint by allowing filtering and sorting of rows.

## Response

| Field                 | Type             | Description                                                           |
| --------------------- | ---------------- | --------------------------------------------------------------------- |
| `starting_token`      | integer          | The starting token used for this request                              |
| `limit`               | integer          | The limit used for this request                                       |
| `paginated`           | boolean          | Whether the results are paginated                                     |
| `next_starting_token` | integer or null  | Token to use for the next page of results, or null if no more results |
| `column_names`        | array of strings | Names of columns in the dataset                                       |
| `rows`                | array of objects | List of dataset row objects matching the query                        |

### Row Object

| Field      | Type           | Description                                                                 |
| ---------- | -------------- | --------------------------------------------------------------------------- |
| `row_id`   | string (UUID)  | Unique identifier for the row                                               |
| `index`    | integer        | Index of the row in the dataset                                             |
| `values`   | array          | Values for each column in the row (can be string, number, integer, or null) |
| `metadata` | object or null | Additional metadata associated with the row                                 |

## Notes

- This endpoint is more powerful than the Get Dataset Content endpoint for complex queries
- You can combine multiple filters to create complex queries
- Use pagination parameters to efficiently retrieve large result sets
- The column_names array provides the names of columns in the same order as the values in each row
- If the dataset doesn't exist or the user doesn't have permission to access it, a 404 error will be returned
