---
title: Update Dataset Version
openapi: patch /datasets/{dataset_id}/versions/{version_index}
---

This endpoint allows you to update metadata for a specific version of a dataset. Currently, you can update the name of the version to make it more descriptive or meaningful.

## Response

The response returns the updated dataset version object with all its fields.

| Field             | Type               | Description                                                             |
| ----------------- | ------------------ | ----------------------------------------------------------------------- |
| `version_index`   | integer            | Unique index number for this version                                    |
| `name`            | string or null     | Updated name for this version                                           |
| `created_at`      | string (date-time) | When this version was created                                           |
| `created_by_user` | object or null     | Information about the user who created this version                     |
| `num_rows`        | integer            | Number of rows in this version of the dataset                           |
| `column_names`    | array of strings   | Names of columns in this version                                        |
| `rows_added`      | integer            | Number of rows added in this version compared to the previous version   |
| `rows_removed`    | integer            | Number of rows removed in this version compared to the previous version |

## Notes

- This endpoint only updates metadata about the version, not the actual content
- Naming versions can help track important milestones or changes in your dataset
- You must have update permission for the dataset to perform this operation
- If the dataset or version doesn't exist, or the user doesn't have permission to update it, a 404 error will be returned
