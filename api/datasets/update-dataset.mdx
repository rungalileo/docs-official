---
title: Update Dataset
openapi: patch /datasets/{dataset_id}
---

This endpoint allows you to update the metadata of an existing dataset, such as its name or column mappings. This is useful for renaming datasets or reorganizing how columns are categorized.

## Response

The response returns the updated dataset object with all its fields.

| Field | Type | Description |
| ----- | ---- | ----------- |
| `id` | string (UUID) | Unique identifier for the dataset |
| `name` | string | Updated name of the dataset |
| `created_at` | string (date-time) | Creation timestamp |
| `updated_at` | string (date-time) | Last update timestamp |
| `project_count` | integer | Number of projects using this dataset |
| `num_rows` | integer | Number of rows in the dataset |
| `column_names` | array of strings | Names of columns in the dataset |
| `permissions` | array of objects | Permissions associated with the dataset |

## Security

This endpoint requires authentication. You can authenticate using:

- An API key in the `Galileo-API-Key` header
- OAuth2 password flow
- HTTP Basic authentication

## Notes

- You must have update permission for the dataset to perform this operation
- Only the fields provided in the request will be updated; omitted fields will remain unchanged
- Column mappings are used to categorize columns for various operations in Galileo
- If the dataset doesn't exist or the user doesn't have permission to update it, a 404 error will be returned
