---
title: Create Dataset
openapi: post /datasets
---

Datasets in Galileo allow you to organize and maintain collections of test cases for evaluating your AI models. This endpoint enables you to create a new dataset by uploading a file containing your data.

## Request Body

| Field | Type | Description |
| ----- | ---- | ----------- |
| `file` | file (binary) | The dataset file to upload. Required. |
| `draft` | boolean | Whether to create the dataset as a draft. Default: false |

## Response

| Field | Type | Description |
| ----- | ---- | ----------- |
| `id` | string (UUID) | Unique identifier for the dataset |
| `name` | string | Name of the dataset |
| `created_at` | string (date-time) | Creation timestamp |
| `updated_at` | string (date-time) | Last update timestamp |
| `project_count` | integer | Number of projects using this dataset |
| `num_rows` | integer | Number of rows in the dataset |
| `column_names` | array of strings | Names of columns in the dataset |
| `permissions` | array of objects | Permissions associated with the dataset |

## Security

This endpoint requires authentication. You can authenticate using:

- An API key in the `Galileo-API-Key` header
- OAuth2 password flow
- HTTP Basic authentication

## Notes

- The uploaded file must be in one of the supported formats (CSV, Feather, or JSONL)
- CSV files should have a header row with column names
- For large datasets, the upload may take some time to process
- After creating a dataset, you can use it in experiments to evaluate model performance
