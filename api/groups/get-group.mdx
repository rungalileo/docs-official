---
title: Get Group
openapi: get /groups/{group_id}
---

This endpoint retrieves detailed information about a specific group in your Galileo account. Groups are organizational units that help you manage access control and resource sharing across projects, datasets, and other Galileo resources. This endpoint provides comprehensive details about the group, including its members, permissions, and associated resources.

## Path Parameters

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `group_id` | string (UUID) | The unique identifier of the group to retrieve |

## Query Parameters

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| `include_members` | boolean | Optional. Whether to include the list of group members in the response (default: true) |
| `include_resources` | boolean | Optional. Whether to include the list of resources associated with the group (default: false) |

## Response

The response contains detailed information about the requested group.

## Error Responses

| Status Code | Description |
| ----------- | ----------- |
| `404` | Group not found |
| `403` | Insufficient permissions to access the group |

## Security

This endpoint requires authentication. You can authenticate using:

- An API key in the `Galileo-API-Key` header
- OAuth2 password flow
- HTTP Basic authentication

## Notes

- You must be a member of the group or have organization-level permissions to access group details
- Group administrators can see all information about the group, including all members and resources
- Regular members may have limited visibility into certain group settings and resources based on their permissions
- Use this endpoint to verify group membership, check resource access, or retrieve group metadata for integration with other systems
- For large groups, the response may be paginated if `include_members` or `include_resources` is set to true
