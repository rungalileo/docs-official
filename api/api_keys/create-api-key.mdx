---
openapi: post /users/api_keys
---

API keys provide a secure way to authenticate your applications with the Galileo API. They are long-lived tokens that can be used instead of username/password credentials. This endpoint allows you to create a new API key with optional restrictions such as an expiration date or limiting access to a specific project.

## Request Body

| Field         | Type               | Description                                                       |
| ------------- | ------------------ | ----------------------------------------------------------------- |
| `description` | string             | Optional description for the API key                              |
| `expires_at`  | string (date-time) | Optional expiry date for the API key in ISO 8601 format           |
| `project_id`  | string (UUID)      | Optional project ID to restrict the API key to a specific project |

### Example Request

```json
{
  "description": "My application API key",
  "expires_at": "2025-04-01T09:00:00Z"
}
```

## Response

| Field          | Type               | Description                                                                |
| -------------- | ------------------ | -------------------------------------------------------------------------- |
| `id`           | string (UUID)      | Unique identifier for the API key                                          |
| `created_at`   | string (date-time) | Creation timestamp                                                         |
| `updated_at`   | string (date-time) | Last update timestamp                                                      |
| `last_used`    | string (date-time) | When the API key was last used (null if never used)                        |
| `project_id`   | string (UUID)      | Project ID if the key is restricted to a specific project (null otherwise) |
| `project_role` | string             | Role within the project if the key is restricted to a specific project     |
| `api_key`      | string             | The actual API key value (only returned once upon creation)                |

### Example Response

```json
{
  "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "created_at": "2023-01-01T12:00:00Z",
  "updated_at": "2023-01-01T12:00:00Z",
  "last_used": null,
  "project_id": null,
  "project_role": null,
  "api_key": "glo_1234567890abcdef1234567890abcdef"
}
```

## Security

This endpoint requires authentication. You can authenticate using:

- An existing API key in the `Galileo-API-Key` header
- OAuth2 password flow
- HTTP Basic authentication

## Notes

- The API key value (`api_key` field) is only returned once upon creation. Make sure to store it securely.
- If you specify a `project_id`, the API key will only have access to that specific project.
- Setting an `expires_at` date is recommended for security reasons.
