---
title: Get Multi-Run Sample IDs
openapi: get /projects/{project_id}/runs/prompts/rows/columnar
---

This endpoint retrieves sample IDs across multiple experiment runs in a columnar format, enabling cross-run analysis and comparison. It's particularly valuable for tracking how the same prompts or inputs perform across different model versions, parameter settings, or prompt templates. The columnar representation makes it easy to align and compare results from multiple runs side by side.

## Response

The response contains sample IDs and optional metadata in a columnar format, organized by run.

## Error Responses

| Status Code | Description                                             |
| ----------- | ------------------------------------------------------- |
| `404`       | Project not found                                       |
| `403`       | Insufficient permissions to access the runs             |
| `400`       | Invalid query parameters or missing required parameters |

## Notes

- This endpoint is particularly useful for A/B testing and comparing the performance of different model configurations
- The `input_hash` field helps identify the same input across different runs, enabling direct comparison
- Use the `filters` parameter to focus on specific subsets of data, such as inputs with certain characteristics
- Including metadata provides additional context for each sample, which can help identify performance factors
- The columnar format makes it easy to import the data into analysis tools for further processing
- For large datasets with many runs, use pagination to manage response size and loading times
- This endpoint pairs well with the "Get Multi-Run Samples for Row IDs" endpoint, which provides the full sample data for detailed analysis
