---
title: Get Rows
openapi: get /projects/{project_id}/runs/{run_id}/prompts/rows
---

This endpoint retrieves a list of rows from a specific experiment run. It provides basic information about each row, including identifiers and timestamps, making it useful for discovering available data before fetching more detailed information. This endpoint supports pagination and filtering to help manage large datasets.

## Response

The response contains a list of rows from the specified experiment run.

## Error Responses

| Status Code | Description                                 |
| ----------- | ------------------------------------------- |
| `404`       | Project or run not found                    |
| `403`       | Insufficient permissions to access the data |
| `400`       | Invalid query parameters                    |

## Notes

- This endpoint provides a lightweight overview of available rows, without including the full prompt and completion data
- Use the `filter` parameter to narrow down the results based on specific criteria
- The `has_metrics` field indicates whether evaluation metrics are available for the row
- The `has_feedback` field indicates whether user feedback is available for the row
- The `prompt_hash` can be used to identify rows with the same input across different runs
- For detailed information about specific rows, use the Get Row as Column endpoint
- For analyzing multiple rows with their full data, use the Get Rows as Columns with Filters endpoint
- For large datasets, use pagination to manage response size and loading times
